# Index Page Template

Write `index.md` after all content pages are complete. Mirror the structure of `apps/8fold/src/index.md`.

## Required sections (in order)

1. **Frontmatter** — `title` and `description`.

2. **H1 + framing paragraph** — what the collection is and who it is for. 2-4 sentences, no signposting.

3. **AI-assistance disclaimer** — one short paragraph:
   > This collection was generated with AI assistance as a synthesis of classical sources and edited for consistency. It is a study aid, not a primary source. Consult the sources listed at the end for authoritative treatments.

4. **How to Read This Collection** — present the reading order: Core groups first (list each group with its Overview pages linked), then note Additional Reading groups as optional depth. Use `[Title](Filename.md)` cross-links for Overview pages.

5. **Working Vocabulary** — a definition list of 8-12 recurring terms. Format each as:
   ```
   **Term** - one-sentence definition
   ```
   Pick terms that appear across multiple pages. Do not include every concept — just the ones a reader will encounter repeatedly and may not know before starting.

6. **Sources and Acknowledgements** — list the sources from RESEARCH.md's Sources section. Group them the same way RESEARCH.md does (e.g., Stanford Encyclopedia, IEP, Britannica, etc.). One line per source: title, URL.

## Content rules

The same §1-§40 rules apply. Particular watch-outs for index pages:
- §28 Signposting — "Let's explore the collection" → cut
- §29 Fragmented headers — don't restate the H1 in the framing paragraph
- §37 Triadic padding — the working vocabulary should be specific, not padded with adjectives
